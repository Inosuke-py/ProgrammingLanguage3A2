# Kino — Project Context & Development Plan

## What is Kino?

Kino is an AI-powered interactive quiz generator. Users upload study materials (PDF, PPTX, DOCX), review them through a live interactive document viewer, and take highly customizable quizzes generated by a **local Ollama instance running Mistral**. The system is gamified, collaborative, and built around closing the learning loop — not just testing knowledge, but explaining *why* answers are right or wrong.

**Key constraint:** All AI processing runs locally via Ollama + Mistral. No data leaves the user's environment.

## Core User Loop

```
Upload Material → Review Interactively → Customize Quiz → Take Quiz → Learn from Mistakes → Track Mastery
```

## Current Build Phase

**Phase 1 — Core MVP** (not yet started)
- PDF upload and parsing
- Mistral (Ollama) integration for MCQ generation
- Basic quiz UI (multiple choice, true/false)
- Score tracking per session

| Layer | Choice |

| Frontend | React + TypeScript |

| Backend | Python + FastAPI |
| Database | PostgreSQL |
| AI Engine | Ollama (local) + Mistral |
| PDF Processing | pdf.js (viewer) + PyMuPDF (parsing) |
| Real-time | Socket.io or WebSockets (Phase 5) |
| Auth | Google OAuth 2.0 |
| File Storage | Local filesystem (dev) → MinIO on VPS |

## Design Principles

1. Close the loop — every wrong answer must teach, not just penalize
2. Local-first AI — Mistral on Ollama, no data leaves the machine
3. Progressive complexity — simple UI that reveals power features as users engage
4. Trust through transparency — always show the source of a generated question
5. Alive, not static — animations and feedback make the quiz feel like a game
6. Mobile-aware from day one

## Key Features (Summary)

- Interactive PDF viewer with hover definitions, click-to-generate questions, difficulty zone highlighting
- Smart quiz customization: difficulty, Bloom's taxonomy, question types, time pressure, weak topic focus
- Advanced question types: MCQ, true/false, fill in blanks, matching, arrange in order, assertion/reason, case studies, scenario questions, diagram labeling
- Quiz modes: Quick Quiz, Deep Exam, Adaptive, Survival, Explain & Learn, Flashcard Conversion, Battle Mode, Study Rooms
- "Why did I get this wrong?" — split-screen with highlighted PDF source + Mistral explanation + "Explain like I'm 12"
- Confidence-based learning (Guessing / Somewhat sure / Very sure) to detect fake mastery
- Gamification: XP, levels, badges, streaks, combos, mastery decay, spaced repetition
- Social: shared material sets, live battle mode, study rooms, class mode with teacher dashboard

## Always Remember

- The full plan lives in `PLAN.md` — refer to it for detailed specs before implementing any feature
- Match implementations to the current phase before jumping ahead
- Mistral via Ollama is the only AI engine — do not introduce external AI API calls
- Every quiz question must be traceable back to a source section in the original document
